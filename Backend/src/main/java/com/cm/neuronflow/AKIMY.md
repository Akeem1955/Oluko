### Corrected Plan (Grouped into 5 Logic Stages + Assessment)

## Stage 1: Identity (Who is learning?)

* **`Users`**: The student.
  *Pipeline role:* **Actor**. Starts everything (login, creating courses, taking quizzes). Holds `availableToken` for AI cost control.
* **`UserLearningStats`**: The scoreboard.
  *Pipeline role:* **Gamification**. Updates after sessions/quizzes (streaks, minutes, rank, lessons completed).

---

## Stage 2: Setup (What are they learning?)

* **`MyClass`**: The course folder.
  *Pipeline role:* **Container**. Groups everything under one class ID. Enforces “max 3 ACTIVE courses”.
* **`ClassMaterial`**: The source of truth.
  *Pipeline role:* **AI Context**. Holds the raw source (topic text / PDF bytes / YouTube URL + meta). AI reads this to generate the syllabus.

---

## Stage 3: Structure (The syllabus + session state)

* **`LessonUnit`**: The chapter/unit.
  *Pipeline role:* **Roadmap**. Generated by the “Architect AI” right after course creation. Breaks material into logical chunks (e.g., Unit 1: Arrays).
* **`ClassSession`**: The classroom session.
  *Pipeline role:* **State Management**. Tracks the active learning run, enforces interruption limits (3), and prevents conflicting concurrent runs.

---

## Stage 4: Brain (Teacher planning + script)

* **`AgentPlan`**: The strategy + plan state.
  *Pipeline role:* **AI Logic**. Generated when a user starts a `LessonUnit`. Stores:

    * the **objective**
    * the **structured plan JSON** (ordered steps, checkpoints, quiz gates, continue triggers)
* **`PlanStep`**: The script line (atomic action).
  *Pipeline role:* **Action**. Each step is executed in order and contains `StepPayload` (what to say/draw/seek/ask). Includes config like `maxRetries`.

*Note:* `ToolRun` is not used—its responsibility is represented inside `PlanStep` via `StepType + StepPayload`.

---

## Stage 5: Runtime (Execution + logs)

* **`StepExecution`**: The log.
  *Pipeline role:* **History & Debugging**. Records what actually happened per step attempt: user transcript, evaluation JSON, state, timestamps. Used to derive retries and outcomes.

---

## Assessment (The test gate)

* **`Quiz`**: The exam paper.
  *Pipeline role:* **Gatekeeper**. Linked to a `PlanStep`. Blocks progress until pass.
* **`QuizQuestion`**: The question bank.
  *Pipeline role:* **Content**. Holds question, options, correct answer index, explanation.
* **`QuizAttempt`**: The report card.
  *Pipeline role:* **Grading**. Links user to quiz, stores score and pass/fail with timestamp.

---

## Summary Pipeline Flow

1. User creates **`MyClass`** + **`ClassMaterial`**.
2. AI (Architect) reads material → generates **`LessonUnit`**.
3. User starts a unit → AI (Teacher) generates **`AgentPlan`** (objective + plan JSON) + **`PlanStep`**.
4. Frontend executes **`PlanStep`** actions (speak/draw/video/ask).
5. Backend records **`StepExecution`** logs (attempts + evaluations).
6. Steps trigger **`Quiz`** → user submits **`QuizAttempt`** → result feeds back into plan continuation or retry logic.
